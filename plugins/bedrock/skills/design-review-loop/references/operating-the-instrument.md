# Operating the Instrument — draws, calibration, generations, attribution

The loop is a nondeterministic measuring instrument, and this file is the discipline for *operating* one: how many draws to buy and when, how to calibrate against known answers, how to control change across instrument generations, how to attribute a shift when one appears, and what the dispositions are for runs that die or streams that alarm. It governs the running of the instrument the SKILL.md defines; it does not touch the loop's convergence semantics, which live in `convergence-machinery.md` and the runner. The audit that scores a run afterward is the SKILL.md's cold post-run audit; this file is what you do before and between runs so that the audit has interpretable data to score.

## Buying recall — draws, runs, and unions

- **The union over independent draws strictly dominates any single draw.** Every run surfaces sharp true findings its siblings miss, while a stable core reproduces everywhere. The operating question is never "can one run be trusted" — it is "how many draws buy the recall this target needs." Convergence and trust criteria designed for single-run semantics are aimed at the wrong object.
- **Single-draw recall collapses with scale.** At distilled-set scale, same-bytes draws overlap ~35–75% by hat; at SDD scale (~100k-token inputs), two same-bytes runs produce near-zero defect overlap, each holding strong finds the other misses entirely. The high-recall operating mode is **union-of-k fresh runs** — and the cost lever that makes k affordable (prompt caching) is therefore the *enabler of the operating mode*, not an optimization.
- **Fresh runs, not repeat passes — and not retries.** A second pass within a run (no document change) yields no new true defects; fresh runs find disjoint ones. And empty-emissions cluster *by run*, not by hat — a run's shared sampling conditions drive collapse across hats simultaneously — so redundancy assumes failure correlation within a run and independence across runs: **k fresh runs beats k retries inside one run.**
- **Draw-level redundancy substitutes for component-level reliability — and diagnoses the failure for free.** A two-draw union absorbs a full reviewer-hat dropout without coverage loss, and the cross-draw comparison itself proves the silent draw was instrument noise rather than document signal ("nothing to attack" refuted by the same hat on the same bytes). Fix the ensemble by sampling, not the worker by redesign.
- **Choose recall strategy by altitude.** Finding stability tracks granularity inversely (EA-class self-overlap ~75%, LAA ~50%, SA ~35%): big, few, enterprise-altitude findings reproduce; fine-grained soundness findings live in a search space each draw samples differently. One draw may suffice for EA-class recall; SA-class wants the union. Variance is a property of the target's search-space size, not a defect of the reviewer.
- **What reproduces is real; what doesn't gets verified, not trusted.** Running the same review twice on identical input separates signal from noise on both sides of the ledger: genuine defect families and genuine credit clusters reproduce across draws; one-off findings are draw-local whether they turn out valid or not. **k=2 buys the baseline; a bounded delta thereafter needs only k=1** — replication purchases the baseline, not every subsequent measurement.
- **A compliant empty is impossible by calibration — exploit that.** The 2-POSITIVE quality floor, ratified for output quality, doubles as a degenerate-output detector: a hat that finds no defects still owes its strongest credits, so `[]` is not a legitimate null — it is malformed-by-calibration, mechanically detectable, and re-drawable. When a degenerate output mode appears, check whether an existing quality constraint already distinguishes it from a legitimate null before building a detector.

## Calibration — known answers before trusted readings

- **Pre-register calibration markers before the run.** Name the specific known defects the instrument should or shouldn't find — written down, outcomes enumerated — so the post-run audit cannot rationalize whatever happens. When you are both experimenter and audience, hindsight rationalization is the default failure mode; pre-registration is the cheap structural fence.
- **Repeated registered misses become mechanism.** A single unregistered miss is noise; the same defect pre-registered and missed across runs from different angles characterizes *where the instrument's attention anchors*. Pre-registration converts misses from embarrassments into instrument data.
- **A deliberately-held known defect is a free positive control.** Retaining one known-true positive through a review cycle converts the run into a detection test with an answer key: detection is a calibration datum, non-detection is a named instrument gap — either way the cycle yields instrument evidence on top of its document verdicts.
- **Calibration targets must match the instrument's doctype.** A calibration run's product is *interpretable evidence*; feed a decision-record reviewer an off-doctype target and the noise cannot be attributed (stance leakage, or a confused stance?). Choose targets for the interpretability of the data they will produce, never for convenience or the appeal of dogfooding.
- **Deferred calibration imposes retention now.** A decision to tune a parameter later obligates persisting, today, the inputs needed to reconstruct the fit — what you intend to tune tomorrow constrains what you must record now, and the first re-tune is when a missing input bites.
- **A constant governs only the population its calibration evidence samples.** A shared tunable is an implicit coupling contract: re-tuning moves every consumer, including ones the evidence never measured. When a second use appears for a calibrated constant, the second use gets a second, decoupled constant — even if initialized equal. Equality of value is a coincidence of today; identity of tunable is a governance commitment about every tomorrow.

## Change control — one change per generation

- **Gen-n is not a bus.** When a generation boundary opens, every backlog change is tempted to ride along. Scope each boundary to exactly one change; the cost of batching is *attribution* — five changes riding one boundary buys five confounded hypotheses for the price of one event. Backlog items keep their own evidence thresholds and wait for their own warrant.
- **Order is content for LLMs — there is no "cosmetic" exemption for prompt bytes.** A reorder that changes zero words still does semantic work (position, recency, framing), so any byte change to a prompt takes the same generation marker as a content change. The neutrality claim is only verifiable empirically, after the fact — and the boundary marker is what makes that empiricism possible.

## Attribution — when a stream shifts

- **Attribution decomposes per stream.** A binary question about an instrument shift ("amendment working vs. corrective over-suppressing") is usually answerable only after decomposing the shift per stream — competing hypotheses are often both partly right about different components and both wrong as totals. Held open for cold eyes, such questions dissolve rather than resolve.
- **Silence after two changes is an attribution problem.** Quiet streams right after two events landed are evidence of healing *or* muting, undecidable from inside the session that made both changes. Two disciplines keep it tractable: pre-commit each event as an attribution candidate *before* the run, and hand the judgment to cold eyes.
- **Precision *should* drop after a good amendment.** A good amendment removes the reproducible valid findings, so the residual attack surface selects for false positives — validity rate is not an instrument-quality metric across an amendment boundary. The honest comparator is what the valid findings would have been, not the rate; cross-hat, cross-draw convergence stays meaningful when rates don't.
- **Plot failure geometry before fixing "flakiness."** Role-correlated failure (same position, same structural pressure, across runs) points at an incentive built into the role's rules; run-correlated failure (multiple roles failing together in one execution, clean in the next) points at a run-level latent factor. Same surface symptom, opposite diagnoses, opposite fixes.

## Run dispositions — deaths, alarms, and absences

- **An aborted run's output is evidence, not waste.** Never splice a dead run into a live one — and never discard what it already bought: same-generation, same-bytes emissions from a dead run make free variance studies possible and can hold unique true findings no completed run has.
- **Chartered watch streams never abort a run; only operational health does.** The phenomena a run is chartered to watch *are* the data — killing a run for exhibiting the thing being measured destroys the measurement. Abort on parse storms, volume swamps, cost runaway; never on the hypothesis showing up.
- **Input absence corrupts — it doesn't degrade.** A conformance reviewer with an empty authorities set doesn't do worse conformance review; it silently becomes an internal-soundness reviewer and corrupts the independence evidence the roster exists to gather. This is *why* empty substrate is the fail-loud prep gate the runner already enforces (see the SKILL.md's load-bearing invariants) — absence is a validity failure, never a degraded mode.
- **A gated actor's zero-fire is tested as the guard working before it is read as the actor broken.** When an actor behind a safety gate never activates, the null is ambiguous: unexercised capability and working safeguard present identically and are opposite verdicts. Test whether the gate is *correctly withholding* (does every real input legitimately trip the gate's condition?) before concluding the actor is broken — the convergence gate refusing to hand an autonomous writer any document still carrying an unresolved decision is the gate doing exactly its job.

## Audit vocabulary — the FALSE-POS diagnosis

The cold post-run audit's validity vocabulary (TRUE / TRUE- / OVER / FALSE-POS / HELD-MIS-SEV) lives in the SKILL.md's cold-audit section; this file adds only the *diagnosis* behind its FALSE-POS class. De-deferencing a reviewer (removing its permission to trust narrated closure) buys re-attack of ratified decisions — expected — but also surfaces findings whose factual predicates are simply false against the reviewed bytes. The mechanism: deference and reading-fidelity were entangled — a reviewer permitted to trust the narrative was implicitly reading more carefully *because* it attacked less; forced aggression trades reading fidelity for coverage. Operating consequence: validity-rate comparisons across instrument generations must separate wrong-reading (FALSE-POS) from over-reach (OVER) — they respond to different fixes.

## What carries evidence, and what is a bet

Split every "the pattern holds" claim by altitude and give each layer its honest scope — a well-evidenced sub-claim must not vouch for an unevidenced super-claim. As of this file's authoring: the **draw/union laws, the calibration disciplines, the generation change-control rules, and the attribution methods above are instrument-level observations** from the one harness family this skill binds to — repeatedly exercised, several independently rediscovered across targets. The **transfer of any of them to a second instrument family is a design bet**, not an observation: the laws are phrased generally because their mechanisms are general, but no second family has run them. That second family is the named climb trigger in this skill's ratified disposition — its arrival re-opens the standalone-empiricism question as its own ratification, never by silent generalization from this file.
